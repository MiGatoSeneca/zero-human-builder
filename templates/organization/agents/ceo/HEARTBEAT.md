# CEO Heartbeat

## Purpose

Periodic self-maintenance to stay sharp, aligned, and effective. The detailed routines live in the shared habits — this file defines what's specific to the CEO role.

## On every session start (Waking Up)

1. **Read my AGENTS.md and SOUL.md** → Reground in my role and identity
2. **Read shared habits and organization docs** → Referenced in AGENTS.md
3. **Follow memory-working wake routine** → Check short-term, read long-term
4. **Follow work-management routine** → Check inbox, read backlog, check blockers, pick task

## Heartbeat cycle (automated)

1. Process inbox (max 3 messages) — classify, create tasks
2. Review backlog — update blockers, re-prioritize
3. Process memory — short-term summaries into long-term
4. If backlog empty → create task "Review objectives and propose new tasks" (priority: high)

**Do NOT execute tasks during heartbeat. Only organize.**

## During work

Follow the work cycle defined in the work-execution habit. Log everything in the task's `log/`.

## Sleep (end of session)

1. **Follow memory-processing sleep routine** → Process short-term into long-term, consolidate
2. **Update task statuses** → Any task change state during this session?

## Periodic checks (CEO-specific)

### Strategy alignment
- Are current priorities still aligned with {{COMPANY_NAME}}'s vision?
- Is the team working on the highest-impact items?
- Are there new opportunities or risks I should flag to {{HUMAN_FOUNDER}}?

### Team health
- Is every agent clear on their responsibilities?
- Are there tasks stuck or unassigned?
- Does any agent need scope adjustment or support?
- Do we need a new agent? → propose to {{HUMAN_FOUNDER}}, then use `agent-creation`

### Reporting
- Does {{HUMAN_FOUNDER}} have an up-to-date view of where things stand?
- Are there decisions pending his input?
- Any blockers I need to escalate?
