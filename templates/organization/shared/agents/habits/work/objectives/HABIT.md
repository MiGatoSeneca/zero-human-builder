# Objectives

Every task should connect to an objective. If it doesn't, question why it exists.

## Where to Find Objectives

```
Company:  organization/drive/objectives/monthly.md
Yours:    drive/objectives/current.md
```

## Objective Format

```markdown
## [Objective name]

- **Metric**: How we measure success — a number
- **Target**: The specific number
- **Deadline**: Date
- **Owner**: Who is accountable
- **Status**: on-track | at-risk | off-track | achieved | missed
- **Contributes to**: Parent objective

### Key Results
- [ ] Specific deliverable — date
```

## Rules

1. **Impact, not activity** — Objectives are outcomes. Tasks go in key results.
2. **Measurable** — Every objective needs a number.
3. **Time-bound** — Every objective has a deadline.
4. **Owned** — One owner.
5. **Connected** — Every agent objective traces to a company objective.
6. **Max 5** per period.

## Proposing New Tasks (when backlog is empty)

1. Read your objectives and company objectives
2. Find gaps — what's not covered by current tasks?
3. Create tasks with status `proposed` for the highest-impact gaps
4. Send `feedback-request` to your manager with the proposals

Each task must connect to an objective and have a clear outcome. Proposed tasks cannot be worked on until approved.

## Prioritizing with Objectives

When two tasks compete, the one serving a higher-priority objective wins.
